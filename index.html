Executive Summary

Agent Team Orchestration (ATO) is a production framework for running multi-agent teams with clear roles, structured task lifecycles, and quality gates. It establishes a Role-based Hierarchy (Orchestrator → Builder → Reviewer → Ops) with formal Handoff Protocols and state-machine-driven Task Lifecycle management. Unlike basic dispatching (one-off task delegation), ATO enables sustained workflows with cross-role verification, decision logging, and async communication.

3. Handoff Protocols

When work passes between agents, the handoff message must include:

1. What
Summary of changes
2. Where
Exact file paths
3. How
Test/verify steps
4. Issues
Known limitations
5. Next
Clear next action

Bad Handoff

"Done, check the files."

Good Handoff

"Built auth module at /shared/artifacts/auth/. Run npm test auth to verify. Known issue: rate limiting not implemented yet. Next: reviewer checks error handling."

Scheduled tasks: Daily standup, task dispatch, health checks. Assign to cheapest reliable agent.

Prevent review fatigue and bias by rotating reviewers across builders.

Common Pitfalls

Spawning without clear output paths

Agent produces great work, but you can't find it. Always specify exact output path in spawn prompt.

No review step = quality drift

"It's a small change, skip review." Do this 3 times and you have compounding errors.

Silent agents create blind spots

Require comments at: start, blocker, handoff, completion. If silent, assume stuck.

Orchestrator doing execution work

The orchestrator routes and tracks — it doesn't build. "Just quickly doing this one thing" = lost oversight.

When NOT to Use ATO

Single-agent setups — Just follow standard AGENTS.md. Team orchestration adds overhead.

One-off task delegation — Use sessions_spawn directly. ATO is for sustained workflows.

Simple question routing — Forwarding to a specialist is a message, not a workflow.

Conclusion

Agent Team Orchestration provides a structured, production-ready framework for multi-agent collaboration. By enforcing role clarity, state-driven task tracking, structured handoffs, and mandatory cross-role reviews, ATO prevents the quality drift and coordination blind spots that plague ad-hoc agent deployments. It's not for every use case — solo agents and one-off tasks don't need it — but for sustained team workflows where agents depend on each other's output, ATO is the backbone of reliable multi-agent systems.
